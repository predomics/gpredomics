1. Overview

gpredomics is a high-performance Rust engine for discovering sparse, interpretable classification models from high-dimensional omics data (metagenomics, transcriptomics, metabolomics). It implements the BTR (Binary-Ternary-Ratio) framework where models use discrete coefficients {-1, 0, +1} that clinicians can directly interpret.

The Problem

Standard machine learning on omics data produces black-box models (Random Forest, XGBoost) that achieve high accuracy but cannot be interpreted by domain experts. Feature selection is computationally expensive, models overfit easily on small clinical cohorts, and results vary across random seeds.

The Solution

gpredomics constrains models to use discrete coefficients, producing rules like:

Class 1: (Bacteroides + Prevotella) - (Faecalibacterium) ≥ 1

This is directly interpretable: if the sum of Bacteroides and Prevotella prevalence minus Faecalibacterium prevalence exceeds 1, predict disease. Seven different optimization algorithms search for these sparse models, and a voting ensemble combines the best ones.

3. Data Pipeline

3.1 Input Format

gpredomics expects tab-separated files with features in rows (transposed omics convention):

# X.tsv (features × samples)
feature_name    sample1    sample2    sample3    ...
msp_0001        0.023      0.001      0.045      ...
msp_0002        0.0        0.012      0.003      ...

# y.tsv (class labels)
sample_name    class
sample1        0
sample2        1
sample3        0

3.2 Feature Selection

Before optimization, features are filtered by statistical significance:

MethodTestUse Case
wilcoxonWilcoxon rank-sumNon-parametric, default for metagenomics
studenttStudent's t-testParametric, normal distributions
bayesian_fisherBayesian + Fisher's exactBayes factor filtering

Filtering criteria (configurable):

  • feature_minimal_prevalence_pct: minimum % of samples with non-zero values (default: 10%)
  • feature_maximal_adj_pvalue: maximum adjusted p-value (default: 0.05)
  • feature_minimal_feature_value: minimum absolute feature value (default: 0.0)

3.3 Data Types

TypeCodeTransformUse Case
RawrawNo transformAbundance data
Prevalenceprevx > 0 → 1, else 0Presence/absence patterns
Logloglog₁₀(x + ε)Log-scale abundance

Multiple data types can be combined: data_type: "raw,prev" generates separate populations for each.

4. Individual & Model Languages

An Individual represents a single sparse classification model. Its key components:

pub struct Individual {
    pub features: BTreeMap<usize, i8>,  // feature_idx → coefficient
    pub k: usize,                        // number of active features
    pub fit: f64,                        // penalized fitness (universal)
    pub cls: ClassificationMetrics,      // AUC, sensitivity, specificity, ...
    pub language: u8,                    // model language
    pub data_type: u8,                   // data transformation
    pub hash: u64,                       // deterministic hash for deduplication
}
BTreeMap for determinism: Features are stored in a BTreeMap (not HashMap) to ensure deterministic iteration order. This guarantees identical results across runs with the same seed, regardless of Rust's random HashMap seeding.

4.1 Model Languages

LanguageCoefficientsScoring FormulaExample
Binary{0, +1}score = Σ(xᵢ) ≥ tA + B + C ≥ 2
Ternary{-1, 0, +1}score = Σ(cᵢ × xᵢ) ≥ tA + B - C ≥ 1
Pow2{...-4,-2,-1,0,+1,+2,+4...}score = Σ(2ⁿ × xᵢ) ≥ t4A + 2B - 8C ≥ 3
Ratio{-1, 0, +1}score = Σ(pos) / Σ(neg) ≥ t(A+B) / (C+D) ≥ 1.5
MCMC Genericcontinuous βP(y=1) = σ(a·pos + b·neg + c)Bayesian posterior

5. Optimization Algorithms

All algorithms share a common interface: they receive preprocessed Data and Param, and return a Vec<Population> of candidate models ranked by fitness.

5.1 Genetic Algorithm (GA)

Population-based Recombinative

The GA evolves a population of models through selection, crossover, and mutation over multiple generations. It is the original and most mature algorithm in gpredomics.

Evolutionary Operators

OperatorDescriptionParameter
Elite selectionTop models preserved unchangedselect_elite_pct: 2.0
Niche selectionBest per ecological niche (language × data_type)select_niche_pct: 0.0
Random selectionRandom survival for diversityselect_random_pct: 2.0
CrossoverUniform crossover of feature setsmutated_children_pct: 80.0
MutationRandom feature add/remove/flipmutated_features_pct: 20.0

Reference: Holland, J.H. (1975). Adaptation in Natural and Artificial Systems. University of Michigan Press.

5.2 Beam Search

Systematic Exhaustive

Beam Search systematically explores model space level by level, from k=1 features to k=k_stop. At each level, it evaluates all combinations (or the best subset) and selects the Family of Best Models (FBM) to extend.

Methods

  • LimitedExhaustive (default): Evaluate all C(n,k) combinations, pruning when too many
  • ParallelForward: Extend existing models by one feature at a time

Reference: Lowerre, B.T. (1976). The HARPY speech recognition system. Carnegie Mellon University.

5.3 MCMC / Bayesian Inference

Bayesian Posterior sampling

Bayesian logistic regression with Metropolis-Hastings sampling. Two modes:

  • Gibbs variable selection (default, method: gibbs): Joint sampling of feature inclusion and coefficients in a single MCMC run, with a sparsity prior P(k) controlling model complexity.
  • Sequential Backward Selection (method: sbs): Nested MCMC at each feature elimination step (original method).

The Gibbs mode includes LASSO pre-screening (top 50 features) and an adaptive BTR model conversion from posterior inclusion probabilities.

References: Metropolis, N. et al. (1953). Equations of state calculations by fast computing machines. J. Chem. Phys.; George, E.I. & McCulloch, R.E. (1993). Variable selection via Gibbs sampling. JASA.

5.4 Ant Colony Optimization (ACO)

Population-based Constructive

Ants construct models feature-by-feature using pheromone trails. Higher pheromone on a feature means it was found useful by previous ants. Uses the Max-Min Ant System (MMAS) variant with pheromone bounds to prevent premature convergence.

Key innovation: pheromone is tracked per (feature, coefficient_sign) pair, encoding not just feature importance but also directional association.

Reference: Dorigo, M. & Stützle, T. (2004). Ant Colony Optimization. MIT Press.

5.5 Simulated Annealing (SA)

Single-solution Perturbative

Starts with a random model and iteratively proposes small changes (add/remove/flip one feature). Improvements are always accepted; worse solutions are accepted with probability e-ΔE/T where T decreases over time (cooling schedule).

Reference: Kirkpatrick, S. et al. (1983). Optimization by simulated annealing. Science, 220(4598).

5.6 Iterated Local Search (ILS)

Single-solution Perturbative

Alternates between local search (greedy hill climbing) and perturbation (random multi-feature changes). The local search finds the nearest local optimum; perturbation escapes it to explore new regions.

Reference: Lourenço, H.R. et al. (2003). Iterated Local Search. In Handbook of Metaheuristics, Springer.

5.7 LASSO / Elastic Net

Direct optimization Convex

L1-regularized regression via coordinate descent along a regularization path (αmax → αmin). Produces the mathematically optimal sparse linear model. Continuous LASSO coefficients are converted to discrete BTR coefficients by taking the sign.

Serves as a mathematical baseline: how well can an optimal sparse linear model do compared to heuristic BTR models?

References: Tibshirani, R. (1996). Regression shrinkage and selection via the lasso. JRSS-B; Friedman, J. et al. (2010). Regularization paths for generalized linear models via coordinate descent. JSS.

6. Population & FBM Selection

A Population is a sorted collection of Individual models. The Family of Best Models (FBM) is selected using confidence interval overlap on the fitness metric.

FBM Selection Algorithm

  1. Compute Wilson score confidence interval for the best model's AUC
  2. Include all models whose AUC overlaps with this interval
  3. Remove clones (identical feature sets, using deterministic hashing)
  4. Enforce diversity (minimum Jaccard dissimilarity between models)

Diversity Metric

Signed Jaccard dissimilarity between two models A and B:

J(A,B) = 1 - |A ∩ B| / |A ∪ B|

where intersection considers both feature index AND coefficient sign.

7. Cross-Validation

Stratified K-fold cross-validation with optional double stratification by class and sample annotation (e.g., study site, batch).

Modes

ModeParametersDescription
Inner CVinner_folds: 5Within-fold model selection
Outer CVouter_folds: 5Performance estimation across folds
Overfit penaltyoverfit_penalty: 0.0Penalize train-test performance gap

The FBM is computed across folds: models that appear consistently across multiple folds are ranked higher.

8. Voting Ensemble

The Jury selects diverse, high-performing models from the FBM and combines their predictions through voting.

Expert Selection Pipeline

  1. FBM CI filter: Models within AUC confidence interval of the best
  2. min_perf filter: Minimum individual AUC (default: 0.5)
  3. min_diversity filter: Minimum Jaccard distance between experts (default: 5%)

Voting Methods

MethodRule
MajorityClass = mode of expert predictions
ConsensusClass = unanimous agreement, else reject

Expert Specialization

Experts can be classified based on their per-class performance:

  • PositiveSpecialist: High sensitivity, moderate specificity → votes only on positive predictions
  • NegativeSpecialist: High specificity, moderate sensitivity → votes only on negative predictions
  • Balanced: Both metrics above threshold → votes on all predictions

10. GPU Acceleration

Optional GPU acceleration via wgpu for fitness evaluation. Supports Metal (macOS), Vulkan (Linux), and DirectX 12 (Windows).

Implementation

The feature matrix X is converted to Compressed Sparse Row (CSR) format. Individual models are encoded as Compressed Sparse Column (CSC) vectors. GPU computes sparse matrix-vector products for all individuals in parallel.

Memory Policies

PolicyBehavior
StrictRespect memory limits strictly, fail if exceeded
AdaptiveBalance memory usage and performance (default)
PerformanceMaximize GPU utilization, may use more memory

12. Configuration Reference

Complete parameter file (param.yaml) with 13 sections:

general:
  seed: 42              # Random seed for reproducibility
  algo: ga              # ga|beam|mcmc|aco|sa|ils|lasso
  language: ter         # bin|ter|pow2|ratio (comma-separated for multi)
  data_type: prev       # raw|prev|log (comma-separated for multi)
  fit: auc              # auc|sensitivity|specificity|mcc|f1|npv|ppv|g_mean
  k_penalty: 0.0001     # Penalty per feature (controls sparsity)
  thread_number: 4      # Parallel threads
  gpu: false            # Enable GPU acceleration
  cv: false             # Enable cross-validation

data:
  X: data/Xtrain.tsv
  y: data/Ytrain.tsv
  Xtest: data/Xtest.tsv
  ytest: data/Ytest.tsv
  features_in_rows: true
  holdout_ratio: 0.2
  feature_selection_method: wilcoxon
  feature_minimal_prevalence_pct: 10
  feature_maximal_adj_pvalue: 0.05

ga:
  population_size: 5000
  max_epochs: 200
  kmin: 1
  kmax: 200

voting:
  vote: true
  method: Majority
  min_perf: 0.5

15. Benchmark Results

Dataset: Qin et al. (2014) — Liver cirrhosis metagenomics. 1,980 features (MSPs), 180 training samples, 30 test samples.

AlgorithmTest AUCModel Size (k)TimeType
🥇Beam Search0.94780.5sSystematic
🥈LASSO / Elastic Net0.882700.1sDirect optimization
🥉Simulated Annealing0.856401sSingle-solution

Interpretation: Beam Search achieves the best test AUC (0.947) with the smallest models (k=8) because it systematically explores all small model combinations. Population-based methods (GA, ACO) find diverse model sets. Single-solution methods (SA, ILS) are the fastest. LASSO provides the mathematical baseline.
